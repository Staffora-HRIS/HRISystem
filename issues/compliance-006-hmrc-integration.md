# [COMPLIANCE] HMRC Integration and PAYE Reporting Not Implemented

**Priority:** CRITICAL
**Labels:** compliance, enhancement
**Effort:** XL

## Description
UK employers must report employee pay and deductions to HMRC in real time via Full Payment Submissions (FPS) and Employer Payment Summaries (EPS). No PAYE, RTI, FPS, or EPS functionality exists. No tax code storage or management, no HMRC API integration, and no P45/P60/P11D generation. While PAYE is typically handled by specialist payroll software, the HRIS should provide data export interfaces, tax code management, and document storage for payroll-generated tax documents.

## Current State
- No PAYE or RTI functionality
- No tax code storage or management
- No HMRC API integration
- No P45/P60/P11D generation logic
- Field registry mentions `national_insurance_number` and `tax_id` but no processing logic
- Documents module supports `tax_form` and `salary_slip` types but no generation

## Expected State
- Payroll data export interface (employee details, hours, absence, statutory pay entitlements)
- Tax code storage and management per employee
- Integration API for payroll systems (Sage, Xero, etc.)
- P45/P60/P11D document storage (generated by payroll system)
- Payroll-relevant data change notifications

## Acceptance Criteria
- [ ] Tax code field added to employee records with validation
- [ ] Payroll data export API endpoint (hours, absence, SSP/SMP/SPP entitlements)
- [ ] P45/P60/P11D document upload and storage via documents module
- [ ] Payroll integration webhook for employee data changes
- [ ] NI category letter tracking
- [ ] Student loan deduction type tracking

## Implementation Notes
This is not about building a payroll system but about providing the data interface that payroll systems need. Create a payroll export module with standardized data output. Most payroll systems accept CSV or API imports. Consider partnerships with UK payroll providers for direct integration.

## Affected Files
- New: `packages/api/src/modules/payroll/` (export routes, service)
- `packages/api/src/modules/hr/repository.ts` (tax code fields)
- New migration for tax code and payroll fields
